## 📁 数据集1：景点景区旅游数据行为分析数据.xlsx

### 🔍 数据结构

**17个字段（列）：**

```
1. tourist_id       - 游客ID（唯一标识）
2. user_nickname    - 用户昵称
3. age              - 年龄
4. gender           - 性别
5. attraction_name  - 景点名称
6. attraction_content - 景点内容描述
7. attraction_type  - 景点类型
8. visit_date       - 游览日期
9. stay_duration    - 停留时长（分钟）
10. ticket_cost     - 门票消费（元）
11. food_cost       - 餐饮消费（元）
12. shopping_cost   - 购物消费（元）
13. transport_cost  - 交通消费（元）
14. entertainment_cost - 娱乐消费（元）
15. total_cost      - 总消费（元）
16. group_size      - 同行人数
17. satisfaction    - 满意度评分
```

### 🎯 主要作用

#### 1. **用户行为分析** ⭐⭐⭐⭐⭐
记录游客的完整游览轨迹和消费行为：
- 在哪些景点停留时间长？
- 不同年龄段游客的偏好是什么？
- 各类消费的占比如何？
- 满意度与哪些因素相关？

**应用示例：**
```python
# 分析热门景点
top_attractions = df.groupby('attraction_name').agg({
    'stay_duration': 'mean',
    'satisfaction': 'mean',
    'total_cost': 'mean'
}).sort_values('stay_duration', ascending=False)

# 输出：
# 灵山大佛: 平均停留45分钟，满意度4.8，人均消费¥280
# 九龙灌浴: 平均停留30分钟，满意度4.6，人均消费¥150
```

#### 2. **个性化推荐引擎** ⭐⭐⭐⭐⭐
基于历史行为数据构建推荐算法：
- **协同过滤**："喜欢灵山大佛的游客也喜欢..."
- **基于内容**："您对佛教文化感兴趣，推荐参观梵宫"
- **基于人群**："25-35岁女性游客常选路线：..."

### 🎯 主要作用

#### 1. **RAG核心知识库** ⭐⭐⭐⭐⭐
这是**事实性问答的主要数据源**，保证准确率≥90%：

**典型问题及检索结果：**
```
Q: "灵山大佛有多高？"
→ 检索到：LS-003 灵山大佛 | 通高88米，其中佛体79米...

Q: "九龙灌浴什么时候表演？"
→ 检索到：每天10:00、11:30、14:00、15:30、16:30各一场...

Q: "梵宫有什么特色？"
→ 检索到：内部装饰采用东阳木雕、敦煌壁画等传统工艺...
```

**为什么适合RAG？**
- ✅ **高度结构化**：每个字段清晰明确
- ✅ **事实准确**：官方提供的标准数据
- ✅ **易于解析**：表格格式便于程序提取
- ✅ **覆盖全面**：17+13=30个核心景点

#### 2. **景点详情查询** ⭐⭐⭐⭐⭐
回答"XX景点在哪里？有什么特色？"等问题：

```python
# RAG检索流程
query = "灵山大佛在哪里？有什么特点？"

# 向量相似度检索 → 找到最相关的片段

    "【灵山大佛】位于景区核心区秦履峰小灵山南麓...",
    "通高88米，是目前世界上最高的释迦牟尼露天青铜立像...",
    "由1560块铜板焊接而成，用铜700余吨..."
]

# LLM生成答案
answer = llm.generate(query, context=retrieved_chunks)
# → "灵山大佛位于景区核心区的小灵山南麓，通高88米，是世界上最高的
#    释迦牟尼露天青铜立像。它由1560块铜板焊接而成，使用了700多吨铜材..."
```

#### 3. **数据标准化** ⭐⭐⭐⭐
统一的格式便于：
- **批量导入**：一键解析所有景点信息
- **向量化索引**：每个景点作为独立的知识块
- **更新维护**：新增景点只需添加一行

**代码示例（rag_service.py第115-134行）：**
```python
elif suffix == ".xlsx":

    # 判断问题类型
    if any(kw in query for kw in ["多少", "哪里", "什么时候", "开放时间"]):
        # 事实性问题 → 优先检索数据集2
        priority_sources = ["灵山胜境 景点结构化数据集"]
    elif any(kw in query for kw in ["为什么", "历史", "典故", "传说"]):
        # 文化类问题 → 优先检索数据集3
        priority_sources = ["灵山胜境：历史、文化"]
    elif any(kw in query for kw in ["推荐", "路线", "怎么玩"]):
        # 推荐类问题 → 结合数据集1的行为数据
        priority_sources = ["景点景区旅游数据行为分析", "个性化游览指南"]
    else:
        # 通用问题 → 全部检索
        priority_sources = None
    
    # 执行检索
    results = await rag_service.retrieve(query, sources=priority_sources)
    return results
```

### 5. **构建混合检索系统**

```python
class HybridRetriever:
    """混合检索器"""
    
    async def retrieve(self, query: str) -> dict:
        """返回多维度检索结果"""
        
        # 1. 向量检索（语义相似）
        vector_results = await self.vector_search(query)
        
        # 2. 关键词检索（精确匹配）
        keyword_results = await self.keyword_search(query)
        
        # 3. 行为数据检索（热门推荐）
        behavior_results = await self.behavior_search(query)
        
        # 4. 融合排序
        final_results = self.rerank(
            vector_results, 
            keyword_results, 
            behavior_results
        )
        
        return {
            "facts": final_results[:3],      # 事实性信息
            "culture": final_results[3:6],   # 文化背景
            "recommendations": final_results[6:]  # 推荐建议
        }
```
