## 🎯 新增需求规范定义（补全缺失功能）

为了完全满足赛题要求并提高系统实用度，需要新增和修复以下功能模块：

### 1. 多轮对话上下文记忆（游客侧）
- **功能描述**: 系统必须记录当前 Session 内的历史对话。在向大模型发送 Prompt 时，应携带最近的数轮对话历史（限制在 5-10 轮以防 Token 超限）。
- **用户体验**: 游客可以进行指代性追问。例如，第一轮问“灵山大佛有多高？”，第二轮可以直接追问“怎么去那里？”或“它是谁建造的？”，AI 数字人能够准确理解“那里”和“它”是指灵山大佛。

### 2. 个性化讲解重点动态微调（游客侧）
- **功能描述**: 
  - 前端游客初次进入系统时，可选择初始兴趣偏好（如“历史人文”、“拍照打卡”、“自然风光”等）。
  - 后端为当前 Session 维护一个**用户画像特征（UserProfile）**，根据游客选择的标签以及对话中提取出的关键词（利用分词或 LLM 提取）动态计算兴趣权重。
  - 当游客询问某个景点时，大模型在生成回复时会结合用户画像，**自动调整讲解词的重点**。例如：
    - **历史偏好者**：重点讲解大佛的历史渊源、玄奘取经小灵山的典故。
    - **拍照打卡偏好者**：重点提供最佳拍摄机位（如“在大照壁前 45 度角是拍大佛全景的最佳点”）、最佳拍摄光线时机。
    - **亲子/休闲偏好者**：重点提供游览便利度信息（如无障碍通道、喂鱼点、萌宠乐园位置）。

### 3. 游客端满意度反馈收集（游客侧）
- **功能描述**: 
  - 在游客聊天界面的侧边栏或对话框内，提供一个“服务评价”悬浮按钮或弹窗。
  - 游客可对本次导览服务进行打分（1-5 星）并填写文字建议。
  - 评价数据提交至后端的 `/api/analytics/feedback` 接口，写入 `visitor_feedback` 数据表中。
  - 后台数据大屏的“平均满意度”、“游客感受度报告”直接读取并解析这些真实数据，结合 `jieba` 分词生成真实的“高频反馈关键词”和“服务改进建议”，告别纯 Mock。

### 4. 音频波形/强度驱动的口型同步（游客侧 - 数字人表现力）
- **功能描述**: 
  - 后端生成语音（TTS）时，可输出音频的音轨数据，或由前端在播放音频时利用 `Web Audio API`（`AnalyserNode`）实时分析音频的**振幅（Volume/Amplitude）和频率**。
  - 根据实时振幅大小，动态计算并改变 CSS 人偶的嘴巴高度（如嘴部高度 `scaleY` 随着音量分贝值在 `0.2` 至 `1.5` 之间实时震荡）。
  - 做到“声音大时嘴张得大，声音小时嘴闭合”，彻底摆脱固定频率的循环说话动画。

### 5. RAG 知识库问答对同步同步与管理（管理后台侧）
- **功能描述**: 
  - 修复 `knowledge_service.py` 缺失 `rag_service.add_qa_pair` 导致的运行时崩溃问题。
  - 在 `RAGService` 中实现 `add_qa_pair(question, answer)`，将新建/编辑的问答对格式化为文本片段后，实时计算 Embedding 向量并同步追加到本地向量存储中。

### 6. 管理后台登录鉴权（管理后台侧）
- **功能描述**: 
  - 对 `/admin` 相关页面实行防非法进入限制。
  - 提供管理员登录页面，后端使用安全密码哈希（如 `bcrypt`）验证，并签发 **JWT (JSON Web Token)**。
  - 所有后台管理 API 路由都需要携带 JWT Token 验证通过方可请求，确保知识库和数字人配置的安全性。

### 7. 响应延迟与 TTS 音质优化（系统性能）
- **功能描述**:
  - **默认高质量在线语音**: 调整 `tts.py` 逻辑，优先调用 `edge-tts`（提供温柔逼真的 `zh-CN-XiaoxiaoNeural`），仅在网络超时或彻底无网时降级到系统自带的 `SAPI5` 机器人声。
  - **常见问答缓存**: 在后端的 Redis 或内存中缓存高频问题及对应的 RAG 检索结果，减少 LLM 重复生成时间。
  - **首字流式输出**: 优化 SSE 接口，实现 LLM 与语音播放的极致流式拼接。

## 📅 下一步功能开发优先级规划

为保证在比赛中拿到高分，建议按以下阶段推进缺失功能的开发：

1. **第一阶段 (P0) - 核心 Bug 修复与基础功能补全**:
   - 修复问答对添加崩溃问题。
   - 重构游客端录音按键，改为按住录音、松手发送的真实交互。
   - 默认采用 Edge-TTS 保证数字人发音的自然度。
   - 接入多轮对话上下文历史，实现连贯问答。

2. **第二阶段 (P1) - 体验与深度功能增强**:
   - 游客端添加“服务评价与反馈”弹框，打通真实反馈数据链。
   - 基于前端 `Web Audio API` 提取播放音频的实时振幅，动态驱动 CSS 卡通嘴型，实现真正的声音-口型同步。
   - 增加登录页面，后端引入 JWT 实现 API 鉴权保护。

3. **第三阶段 (P2) - 差异化亮点突破**:
   - 实现个性化偏好选择与讲解重点自动微调（历史/打卡/休闲）。
   - 优化响应时延，引入问答缓存和异常友好提示。
